## 1. 语言风格特点

### 1.1 正式程度

#### 1.1.1 避免口语化词汇列表

| 口语化表达 | 学术化替换 | 示例 |
|-----------|-----------|------|
| 很多 | 大量、显著、可观 | 实验结果表明，该方法在大量样本上表现优异 |
| 很好 | 显著、有效、优异 | 该方法取得了显著的性能提升 |
| 不好 | 欠佳、有限、不足 | 传统方法的检测精度欠佳 |
| 看一下 | 分析、考察、研究 | 本文深入分析了该问题 |
| 做一下 | 开展、进行、实施 | 开展了大量对比实验 |
| 挺多的 | 较为丰富、数量可观 | 数据集包含较为丰富的场景 |
| 还可以 | 具有一定优势、表现良好 | 该方法在特定场景下表现良好 |
| 差不多 | 相近、近似、趋于一致 | 两种方法的性能相近 |
| 说白了 | 换言之、本质上 | 换言之，该问题可转化为优化问题 |
| 挺重要 | 至关重要、具有重要意义 | 特征提取对模型性能至关重要 |
| 挺快 | 高效、快速、收敛迅速 | 所提算法具有较快的收敛速度 |
| 有点慢 | 效率有限、耗时较长 | 传统方法在处理大规模数据时效率有限 |
| 用了一下 | 采用、应用、引入 | 本文采用了深度残差网络 |
| 试了一下 | 验证、探究、评估 | 通过实验验证了所提方法的有效性 |
| 结果还行 | 结果令人满意、达到预期 | 实验结果达到了预期目标 |
| 基本上 | 总体、大体、原则上 | 该方法总体上优于对比方法 |
| 总的来说 | 综上所述、总体而言 | 综上所述，所提方法具有明显优势 |
| 也就是说 | 即、换言之、亦即 | 该过程即特征提取阶段 |
| 其实 | 事实上、实际上 | 事实上，该假设在复杂场景下不成立 |
| 当然 | 显然、无疑、毋庸置疑 | 显然，模型复杂度与计算量正相关 |

#### 1.1.2 避免网络用语与缩写

| 不规范 | 规范表达 |
|-------|---------|
| OK | 可行、有效、满足要求 |
| vs | 与、对比、相较于 |
| etc. | 等、等等（用于列举） |
| i.e. | 即、也就是 |
| e.g. | 例如、如 |
| AI | 人工智能（首次出现需全称） |
| CV | 计算机视觉（首次出现需全称） |
| DL | 深度学习（首次出现需全称） |
| SOTA | 当前最优性能 |

### 1.2 常用学术连接词

#### 1.2.1 因果关系

| 连接词 | 使用场景 | 示例 |
|-------|---------|------|
| 因此 | 直接结论 | 该特征具有较强的判别性，因此可有效区分不同类别 |
| 因而 | 书面化因果 | 模型参数量大幅减少，因而推理速度显著提升 |
| 从而 | 递进因果 | 通过引入注意力机制，从而增强了特征表达能力 |
| 故 | 简洁因果（公式后常用） | 由式(3)可知，该函数单调递增，故存在唯一解 |
| 由此可见 | 总结性因果 | 由此可见，所提方法具有较好的泛化能力 |
| 鉴于此 | 基于前述原因 | 鉴于此，本文提出了一种改进的特征融合策略 |
| 由于 | 前置原因 | 由于卷积神经网络具有强大的特征提取能力... |
| 鉴于 | 正式原因引导 | 鉴于上述问题，本文开展了相关研究 |

#### 1.2.2 转折关系

| 连接词 | 使用场景 | 示例 |
|-------|---------|------|
| 然而 | 标准转折 | 深度学习方法性能优异，然而其计算复杂度较高 |
| 但是 | 口语化，慎用 | 建议替换为"然而""但" |
| 但 | 简洁转折 | 该方法检测精度较高，但实时性不足 |
| 尽管如此 | 让步转折 | 尽管如此，小目标检测仍面临诸多挑战 |
| 虽然...但是... | 标准让步 | 虽然深度网络性能优异，但是其可解释性有限 |
| 反之 | 反向对比 | 若阈值设置过高，则召回率下降；反之，则精确率降低 |
| 相反 | 对立对比 | 与文献[12]的结论相反，本文发现... |

#### 1.2.3 递进关系

| 连接词 | 使用场景 | 示例 |
|-------|---------|------|
| 此外 | 补充并列 | 此外，所提方法还具有较强的鲁棒性 |
| 另外 | 次要补充 | 另外，本文还探讨了不同参数设置的影响 |
| 而且 | 口语化，慎用 | 建议替换为"并且""同时" |
| 并且 | 并列递进 | 该方法检测精度高，并且计算效率优异 |
| 同时 | 同步进行 | 同时，本文还验证了方法在其他数据集上的有效性 |
| 进一步 | 深入阐述 | 进一步分析表明，该模块对性能提升具有关键作用 |
| 在此基础上 | 基于前文 | 在此基础上，本文设计了多尺度特征融合模块 |
| 更为重要的是 | 强调递进 | 更为重要的是，所提方法具有良好的可扩展性 |

#### 1.2.4 顺序关系

| 连接词 | 使用场景 | 示例 |
|-------|---------|------|
| 首先...其次...再次...最后 | 分点论述 | 首先介绍网络结构，其次阐述损失函数，再次分析训练策略，最后给出实验结果 |
| 第一...第二...第三 | 条目列举 | 本文的主要贡献如下：第一，...；第二，...；第三，... |
| 接着 | 过程衔接 | 接着，对特征图进行上采样操作 |
| 随后 | 时间先后 | 随后，将融合后的特征送入检测头 |
| 最终 | 结果总结 | 最终，模型在测试集上取得了优异的性能 |

### 1.4 时态规范

**使用场景：**
- 描述本文完成的实验

- 说明数据收集过程
- 描述已完成的训练过程

## 2. 结构规范

### 2.1 摘要的"四要素"结构

#### 2.1.1 结构要求

摘要字数：200-300字（中文核心期刊通常要求）

**四要素及占比：**
- 目的（20%）：研究背景与问题
- 方法（35%）：本文提出的方法
- 结果（30%）：主要实验结果
- 结论（15%）：研究价值与意义

#### 2.1.2 标准模板

```
【目的】针对[具体问题]，[现有方法的不足]，本文开展了相关研究。
【方法】提出了一种基于[核心技术]的[方法名称]。首先，[模块1的功能]；
其次，[模块2的功能]；最后，[模块3的功能]。
【结果】在[数据集]上的实验结果表明，所提方法的[评价指标]达到[数值]，
相较于[对比方法]提升了[百分比]。
【结论】所提方法能够有效[解决什么问题]，具有[什么优势]。
```

#### 2.1.3 优秀示例

> 针对复杂场景下窨井盖检测精度低、小目标漏检率高的问题，本文提出了一种基于改进YOLOv11的窨井盖检测方法。首先，设计了一种混合残差注意力模块，通过通道-空间双重注意力机制增强特征表达能力；其次，引入广义Dice损失函数优化边界区域学习；最后，构建多尺度特征融合策略提升小目标检测性能。在自建窨井盖数据集上的实验结果表明，所提方法的平均精度均值(mAP)达到87.6%，相较于基线YOLOv11提升4.2个百分点，小目标检测召回率提升8.5%。所提方法能够有效提升复杂场景下的窨井盖检测性能，满足实际工程应用需求。

#### 2.1.4 关键词规范

- 数量：4-6个
- 要求：涵盖研究对象、核心方法、关键技术
- 格式：用分号隔开

**示例：**
目标检测；YOLOv11；注意力机制；窨井盖检测；深度学习；计算机视觉

### 2.2 引言的"倒金字塔"结构（5段式）

#### 2.2.1 结构框架

```
第1段：大背景（研究领域的重要性）
第2段：小背景（具体问题的研究现状）
第3段：问题与不足（现有方法的局限性）
第4段：本文工作（研究内容与主要贡献）
第5段：论文结构（后续章节安排）
```

#### 2.2.2 各段写作要点

**第1段：大背景（约150字）**
- 阐述研究领域的重要性和应用价值
- 引用权威文献或统计数据
- 引出研究主题

**示例开头：**
> 目标检测是计算机视觉领域的核心任务之一，广泛应用于智能交通、安防监控、工业检测等场景[1-3]。随着深度学习技术的发展，基于卷积神经网络的目标检测方法取得了显著进展，检测精度和速度均得到大幅提升[4,5]。

**第2段：小背景（约200字）**
- 聚焦到具体研究问题
- 综述相关技术的发展历程
- 分类介绍代表性方法

**示例结构：**
> 现有目标检测方法主要分为两类：两阶段检测器和单阶段检测器。两阶段方法以Faster R-CNN[6]为代表...单阶段方法以YOLO系列[7-9]为代表...近年来，注意力机制[10]被引入目标检测领域...

**第3段：问题与不足（约150字）**
- 指出现有方法的具体缺陷
- 分析问题产生的原因
- 强调研究的必要性

**示例表述：**
> 然而，现有方法在[特定场景]仍面临以下挑战：1）[问题1]，导致[后果1]；2）[问题2]，使得[后果2]。此外，[补充问题]。因此，开展[具体研究]具有重要的理论意义和应用价值。

**第4段：本文工作（约200字）**
- 概括本文的核心方法
- 分点阐述主要贡献
- 强调创新点

**示例结构：**
> 针对上述问题，本文提出了一种[方法名称]。主要贡献包括：1）[贡献1]；2）[贡献2]；3）[贡献3]。与现有方法相比，所提方法具有[优势1]和[优势2]。

**第5段：论文结构（约50字）**
- 简述后续章节安排
- 每章用一句话概括

**示例：**
> 本文后续章节安排如下：第1节介绍相关工作；第2节详细阐述所提方法；第3节给出实验设置与结果分析；第4节总结全文并展望未来工作。

### 2.3 方法部分的"定义→原理→实现"三段式

#### 2.3.1 整体框架

```
2.1 整体框架
    - 网络结构总览
    - 各模块功能概述
    - 数据流向说明

2.2 [核心模块1]
    2.2.1 问题定义
    2.2.2 设计原理
    2.2.3 具体实现

2.3 [核心模块2]
    ...

2.4 损失函数
    - 各损失项定义
    - 总损失函数
```

#### 2.3.2 问题定义

**要素：**
- 数学符号定义
- 输入输出说明
- 优化目标描述

**示例：**
> 设输入图像为$I \in \mathbb{R}^{H \times W \times 3}$，目标检测结果为$B = \{b_1, b_2, ..., b_n\}$，其中$b_i = (x_i, y_i, w_i, h_i, c_i)$表示第$i$个检测框的中心坐标、宽高和类别。本文的目标是设计一个检测网络$\mathcal{F}$，使得：
> $$\mathcal{F}(I; \theta) = \hat{B}$$
> 其中$\theta$为网络参数，$\hat{B}$为预测结果。

#### 2.3.3 设计原理

**要素：**
- 理论依据
- 设计动机
- 解决思路

**示例：**
> 传统特征提取网络在传播过程中存在信息丢失问题，导致深层特征对小目标的表征能力减弱。为此，本文设计了跨层特征融合模块，通过引入跳跃连接将浅层细节信息与深层语义信息相结合，从而增强多尺度特征表达能力。

#### 2.3.4 具体实现

**要素：**
- 网络结构图
- 算法流程
- 关键代码/公式

**示例结构：**
> 该模块的具体实现如下：首先，...；其次，...；最后，...。算法1给出了该模块的伪代码实现。

### 2.4 实验部分的"设置→结果→分析"结构

#### 2.4.1 实验设置

**必含内容：**

1. **数据集**
   - 数据来源与规模
   - 数据划分方式
   - 数据预处理方法

2. **评价指标**
   - 精确率(Precision)
   - 召回率(Recall)
   - 平均精度均值(mAP)
   - F1分数
   - 推理速度(FPS)

3. **实现细节**
   - 硬件环境
   - 软件框架
   - 超参数设置

4. **对比方法**
   - 选取原则
   - 方法列表
   - 复现方式

#### 2.4.2 实验结果

**必做实验：**

1. **主实验（与SOTA对比）**
   - 定量结果表格
   - 定性结果图示

2. **消融实验**
   - 模块有效性验证
   - 各组件贡献分析

3. **参数敏感性分析**
   - 关键超参数影响
   - 最优参数选取

4. **扩展实验**
   - 不同数据集验证
   - 跨场景泛化测试

#### 2.4.3 结果分析

**分析维度：**
- 数值对比分析（提升幅度、统计显著性）
- 可视化分析（特征图、注意力图、检测示例）
- 失败案例分析（漏检、误检原因）
- 计算效率分析（参数量、计算量、推理速度）

## 3. 数学公式规范

### 3.1 符号命名规范

#### 3.1.1 斜体使用规则

**使用斜体（LaTeX默认）：**
- 变量：$x$, $y$, $\alpha$, $\beta$
- 常量：$c$, $k$, $\lambda$
- 函数参数：$f(x)$, $g(y)$
- 随机变量：$X$, $Y$, $Z$

**示例：**
```latex
设输入特征图为 $x \in \mathbb{R}^{C \times H \times W}$
```

#### 3.1.2 正体使用规则

**使用正体（\mathrm{}或特定命令）：**
- 微分算子：$\mathrm{d}x$, $\mathrm{d}y$
- 数学常数：$\mathrm{e}$, $\mathrm{i}$, $\pi$
- 函数名：$\max$, $\min$, $\exp$, $\log$, $\sin$, $\cos$
- 单位：$\mathrm{km}$, $\mathrm{ms}$, $\mathrm{dB}$
- 缩写：$\mathrm{ReLU}$, $\mathrm{IoU}$, $\mathrm{mAP}$

**示例：**
```latex
损失函数定义为 $L = -\sum_{i=1}^{N} \log(p_i)$
交并比计算公式为 $\mathrm{IoU} = \frac{|A \cap B|}{|A \cup B|}$
```

#### 3.1.3 粗体使用规则

**使用粗体（\mathbf{}或\bm{}）：**
- 向量：$\mathbf{x}$, $\mathbf{y}$, $\mathbf{b}$
- 矩阵：$\mathbf{W}$, $\mathbf{H}$, $\mathbf{F}$

**示例：**

### 3.2 公式编号规则

#### 3.2.1 编号格式

### 3.3 矩阵/向量表示规范

#### 3.3.1 向量表示

## 4. 图表规范

### 4.1 图片分辨率要求

#### 4.1.1 分辨率标准

| 图片类型 | 最低分辨率 | 推荐分辨率 | 格式 |
|---------|-----------|-----------|------|
| 线框图/流程图 | 300 dpi | 600 dpi | EPS, PDF, PNG |
| 照片/实验结果 | 300 dpi | 300-600 dpi | TIFF, PNG, JPG |
| 曲线图/统计图 | 300 dpi | 600 dpi | EPS, PDF, PNG |
| 屏幕截图 | 原始分辨率 | 原始分辨率 | PNG |

#### 4.1.2 尺寸规范

| 图片类型 | 单栏宽度 | 双栏宽度 | 高度比例 |
|---------|---------|---------|---------|
| 单栏图 | 8.0 cm | - | 高度≤24 cm |
| 双栏图 | - | 17.0 cm | 高度≤24 cm |
| 通栏图 | - | 17.0 cm | 高度≤24 cm |

#### 4.1.3 字体规范

- 图中文字：Times New Roman或宋体
- 字号：8-10 pt（与正文协调）
- 坐标轴标签：不小于8 pt
- 图例文字：不小于7 pt

### 4.2 图表标题格式

#### 4.2.1 图题格式

**格式规范：**
- 编号："图1""图2"（阿拉伯数字）
- 空格：编号后空一格
- 位置：图下方居中
- 字体：宋体/黑体，字号略小于正文

**正确示例：**
```
图1 所提方法的网络结构
图2 不同方法的PR曲线对比
图3 注意力权重可视化结果
```

**错误示例：**
```
图 1 所提方法的网络结构（编号后多余空格）
图1. 所提方法的网络结构（使用英文句点）
Figure 1 网络结构（中英混用）
图一 网络结构（使用中文数字）
```

#### 4.2.2 表题格式

**格式规范：**
- 编号："表1""表2"（阿拉伯数字）
- 空格：编号后空一格
- 位置：表上方居中
- 字体：宋体/黑体，字号略小于正文

**正确示例：**
```
表1 不同方法的性能对比
表2 消融实验结果
表3 超参数敏感性分析
```

### 4.3 坐标轴标签规范

#### 4.3.1 标签内容

**必须包含：**
- 物理量名称（中文或英文）
- 单位（用括号括起）

**正确示例：**
```
横轴：迭代次数（次）
横轴：Epoch
纵轴：准确率（%）
纵轴：mAP@0.5 (%)
纵轴：损失值
纵轴：处理时间（ms）
```

**错误示例：**
```
横轴：x（无物理意义）
纵轴：accuracy（未给出单位）
横轴：时间/s（格式不规范）
```

#### 4.3.2 刻度规范

- 刻度值应均匀分布
- 避免过多或过少的刻度
- 科学计数法用于极大/极小值
- 对数坐标需明确标注

### 4.4 图例位置要求

#### 4.4.1 推荐位置

| 位置 | 适用场景 | 说明 |
|-----|---------|------|
| 图内右上角 | 曲线较少时 | 不遮挡数据 |
| 图内右下角 | 曲线从左下开始 | 不遮挡数据 |
| 图内左上角 | 曲线从右下开始 | 不遮挡数据 |
| 图下方 | 曲线较多时 | 横向排列 |
| 图右侧 | 纵向排列 | 适合对比组多 |

#### 4.4.2 图例格式

**格式规范：**
- 符号在前，文字在后
- 各条目间距均匀
- 与图中线条/标记一致

**示例：**
```
— 所提方法
- - - YOLOv8
··· Faster R-CNN
○○○ 文献[12]方法
```

### 4.5 表格规范

#### 4.5.1 三线表格式

**标准格式：**
```
表1 不同方法的性能对比

方法          精确率/%    召回率/%    mAP@0.5/%    FPS
——————————————————————————————————————————————————————
Faster R-CNN    82.3       78.5        80.1        15
YOLOv5          85.6       83.2        84.3        45
YOLOv8          87.1       85.4        86.2        52
所提方法        89.5       87.8        88.6        48
——————————————————————————————————————————————————————
注：加粗表示最优结果。
```

#### 4.5.2 表格内容规范

- 数据对齐：数值右对齐，文本左对齐
- 小数位数：保持一致，通常为1-2位
- 单位：统一标注在表头
- 最优结果：加粗表示
- 缺失数据：用"—"或"N/A"表示

```
深度学习在目标检测领域取得了显著进展。[1]（引用在句号后）
```

| 42 | 图表与正文不符 | 数据不一致 | 核对并修正 |

#### 6.1.5 参考文献错误（43-50）

| 序号 | 错误表达 | 错误类型 | 修正建议 |
|-----|---------|---------|---------|
| 43 | 引用在句号后 | 位置错误 | 放在句号前 |
| 44 | 作者姓名格式混乱 | 格式错误 | 按规范统一 |
| 45 | 缺少文献类型标识 | 格式错误 | 补充[J]/[C]等 |
| 46 | 页码范围错误 | 格式错误 | 使用起止页码 |
| 47 | 网址链接失效 | 可访问性差 | 更新有效链接 |
| 48 | 正文中引用未列出 | 遗漏 | 补充到参考文献 |
| 49 | 参考文献未在正文引用 | 多余 | 删除或补充引用 |
| 50 | 文献排序混乱 | 格式错误 | 按引用顺序排列 |
